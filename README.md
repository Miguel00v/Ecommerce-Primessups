Briefing
1. Introdução
Este projeto final consiste na criação de uma loja virtual focada em produtos de
suplementação. A escolha desse tema reflete a minha valorização pessoal pela
saúde e vitalidade. Na fase de planejamento, decidi utilizar a metodologia de Design
Thinking para garantir uma abordagem centrada no usuário e inovadora. Para o
desenvolvimento, utilizei código puro (HTML, CSS, JavaScript e PHP), com o auxílio
de algumas frameworks e bibliotecas para otimizar o desenvolvimento e garantir
uma experiência de usuário de alta qualidade.
2. Empatia/Público alvo
A loja virtual será direcionada para um público que compartilha interesses em saúde
e bem-estar, particularmente em suplementação. Para definir a persona, realizei
várias pesquisas em vários websites de suplementação, tais como a prozis,
zumub, myprotein, entre outros, onde comparei a navegação, design e
funcionalidades dos vários websites e também as frustrações, percepções e desejos
dos utilizadores.
Assim, para a persona do meu website defini um público-alvo que possui as
seguintes características:
● Idade: 18 a 45 anos
● Gênero: Homens e mulheres
● Interesses: Fitness, musculação, desporto, bem-estar, saúde

Prototipagem/Desenvolvimento
1. Introdução
Após concluir a fase de ideação, iniciei a fase de prototipagem, na qual desenvolvi
meu protótipo utilizando diversas linguagens de programação, como HTML, CSS,
JavaScript e PHP. Além disso, utilizei várias frameworks, bibliotecas e APIs,
incluindo phpMailer, Stripe Payments, entre outras.
2. Base de dados
Primeiramente, iniciei o desenvolvimento da minha base de dados, utilizando o
esboço pré-criado na fase anterior como referência e implementando-a através do
phpMyAdmin com scripts SQL.
Passos Seguidos no Desenvolvimento da Base de Dados:
1. Análise do Esboço:
○ Revisei o esboço da base de dados criado durante a fase de ideação
para entender melhor a estrutura necessária e os requisitos funcionais.
2. Definição do Modelo de Dados:
○ Entidades e Atributos: Identifiquei as entidades principais (como
usuários, produtos e transações) e seus respectivos atributos (como
nome, email, preço e data).
○ Relacionamentos: Defini os relacionamentos entre as entidades,
garantindo que a integridade referencial fosse mantida.
3. Criação do Esquema:
○ Utilizei uma ferramenta de modelagem de banco de dados para criar o
esquema, facilitando a visualização das tabelas e dos
relacionamentos.
○ Tabelas: Criei as tabelas necessárias no banco de dados, definindo os
tipos de dados apropriados para cada campo.
○ Chaves Primárias e Estrangeiras: Estabeleci chaves primárias para
identificar unicamente cada registro e chaves estrangeiras para manter
os relacionamentos entre tabelas.
4. Normalização:
○ Apliquei as formas normais necessárias para evitar redundância de
dados e garantir a consistência da base de dados.
5. Implementação:
○ SQL: Escrevi scripts SQL para criar as tabelas e inserir dados iniciais,
configurando permissões e restrições adequadas.
○ phpMyAdmin: Utilizei o phpMyAdmin para executar os scripts SQL e
gerenciar a base de dados.
6. Testes e Validação:
7
○ Realizei testes para garantir que a base de dados estava funcionando
corretamente, inserindo, atualizando e excluindo dados para verificar a
integridade e consistência.
7. Documentação:
○ Documentei a estrutura da base de dados, incluindo diagramas e
descrições das tabelas e relacionamentos, para facilitar a
compreensão e a manutenção futura.
Esse processo me permitiu desenvolver uma base de dados robusta e bem
estruturada, pronta para suportar as operações do meu protótipo.
3. Back-end
Em seguida, iniciei o desenvolvimento da estrutura de cada página utilizando HTML,
definindo elementos como cabeçalhos, parágrafos, imagens, formulários e botões.
Para implementar a funcionalidade dinâmica e a lógica de processamento,
desenvolvi o back-end dessas páginas com PHP.
Além disso, criei um sistema de checkout utilizando a API do Stripe e implementei
funcionalidades de carrinho de compras e botões para adicionar produtos ao
carrinho.
Passos Seguidos no Desenvolvimento do Back-end:
1. Estruturação das Páginas com HTML:
○ Elementos: Criei a estrutura das páginas web utilizando HTML,
incluindo cabeçalhos, parágrafos, imagens, formulários e botões.
2. Implementação da Lógica de Back-end com PHP:
○ Processamento de Formulários: Desenvolvi scripts PHP para
processar dados enviados por formulários, realizando validações e
armazenamentos de informações no banco de dados.
○ Gerenciamento de Sessões: Implementei a gestão de sessões para
autenticação de utilizadores e controlo de acesso. E uma verificação
de email, na criação de conta, em que depois de um utilizador criar a
sua conta, é gerado um token, esse token é armazenado na base de
dados e é enviado um link com o token, para o email inserido pelo
utilizador, em que depois de clicado a conta fica ativa/disponível de ser
acessada.
○ Integração com a Base de Dados: Utilizei PHP para conectar as
páginas ao banco de dados, permitindo operações como recuperação,
inserção, atualização e exclusão de dados.
○ Configuração da API: Configurei a API do Stripe para permitir
pagamentos online seguros. Isso incluiu a integração de métodos de
pagamento, garantindo que os dados dos usuários fossem
8
armazenados de maneira segura e que os métodos de pagamento
pudessem ser guardados para futuras transações.
3. Criação do Formulário de Checkout:
○ Desenvolvimento do Checkout: Criei um formulário de checkout que
exibe informações pré-definidas pelo usuário, como método de
pagamento, dados fiscais e endereço de entrega. Caso o usuário não
tenha métodos de pagamento ou endereços de entrega armazenados
ou pré-definidos, ele será impedido de concluir a compra até que
essas informações sejam fornecidas.
4. Processamento de Pagamentos:
○ Implementação da Lógica de Pagamento: Desenvolvi a lógica para
processar pagamentos utilizando a API do Stripe, garantindo que
todas as transações fossem executadas de forma segura e precisa.
○ Gerenciamento de Respostas: Implementei a manipulação das
respostas da API do Stripe, verificando a confirmação de pagamentos
e tratando quaisquer erros ou problemas que possam ocorrer durante
o processo.
○ Guardar métodos de pagamento: Para permitir que os utilizadores
guardem os seus métodos de pagamento de forma segura, utilizei a
API do Stripe. Com esta abordagem, é gerado um token de
pagamento que é armazenado de forma segura pelo Stripe.
5. Implementação do Carrinho de Compras:
○ Criação do Carrinho: Desenvolvi um sistema de carrinho de compras
que permite aos usuários adicionar produtos, visualizar itens no
carrinho e ajustar as quantidades dos produtos e removê-los.
○ Utilização da Base de Dados: Os itens do carrinho são armazenados
na base de dados para garantir que as informações sejam
preservadas entre sessões e que o carrinho possa ser recuperado
posteriormente.
○ Tabela de Itens do Carrinho: Desenvolvi uma tabela específica,
chamada itens_carrinho, na base de dados para armazenar os
detalhes dos itens do carrinho. Esta tabela contém colunas para o
produtoID, o carrinhoID e a quantidade de cada item, associando cada
item a um carrinho específico.
○ Adição de Itens: Quando um usuário adiciona um item ao carrinho, o
sistema registra o produtoID e o carrinhoID na tabela itens_carrinho.
○ Atualização e Remoção: O sistema também gerencia atualizações,
como alterações na quantidade de itens ou remoção de produtos,
atualizando na tabela itens_carrinho a coluna quantidade conforme o
necessário.
6. Botão de Adicionar Produtos:
9
○ Funcionalidade do Botão: Implementei botões "Adicionar ao

○ Atualização do Carrinho: Assegurei que o carrinho fosse atualizado

7. Implementação de Funcionalidades Dinâmicas:
○ Interatividade: Adicionei funcionalidades dinâmicas às páginas, como

8. Checkout confirmado:
○ Verificação da Transação: Após a confirmação bem-sucedida do
pagamento, o sistema valida a transação com a API do Stripe para
garantir que o pagamento foi processado corretamente.
○ Geração da Fatura: Utilização da Biblioteca domPDF: Para criar a
fatura, utilizei a biblioteca domPDF. Esta biblioteca permite gerar
documentos PDF dinâmicos a partir de HTML e CSS.
○ Conteúdo da Fatura: A fatura gerada inclui detalhes completos da

fornecida pelo usuário, lista detalhada dos produtos adquiridos, valor
total da compra.
○ Formatação e Design: O design da fatura foi formatado de forma a

○ Envio da Fatura: A fatura é enviada para o email do usuário,
garantindo que ele receba uma confirmação oficial da transação e
detalhes da compra.
○ Página de Confirmação: Após a confirmação do pagamento e o envio
da fatura, o usuário é redirecionado para uma página de confirmação
de compra.
○ Exibição dos Detalhes da Encomenda: A página de confirmação
